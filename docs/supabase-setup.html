GameUp Command Center v5 部署指南

适用阶段：数据库已经导入，下一步要把 GitHub Pages 前端、Supabase Edge Function 后端、Supabase Auth 和 Postgres 数据库打通。

目录
架构 文件放哪里 Supabase SQL Auth 用户 Edge Function 前端配置 本地测试 GitHub Pages 验收测试 排错

0. 这版到底怎么打通？

GitHub Pages 静态前端
  apps/command-center/index.html / src/*.js / styles/*.css
        ↓ 登录：Supabase Auth
        ↓ 请求：/functions/v1/gameup-api
Supabase Edge Function 后端
  校验 access_token
  校验 public.app_users 白名单
  调用 PostgreSQL RPC
        ↓
Supabase Postgres
  characters / parties / game_versions / resources 等表

关键点：前端不直接管理十几张表。前端只提交业务动作，例如 saveCharacter。真正写多表的逻辑放在 PostgreSQL JSONB RPC 里；鉴权和安全边界放在 Edge Function 里。

2. Supabase SQL：先检查，再安装后端函数

2.1 跑结构检查

打开 Supabase：

Supabase Dashboard
→ 你的项目 GameUpCommandCenter
→ 左侧 SQL Editor
→ New query

把这个文件全部复制进去并运行：

supabase/sql/01-check-schema.sql

结果看 status：

结果意思处理
OK表和必要列基本对得上继续下一步
MISSING_COLUMNS你微调后的列名和前端后端预期不同看 missing_columns，把列名告诉我或手动改 SQL
MISSING_TABLE某张核心表不存在确认你是否导入了该模块

2.2 跑后端 RPC + 白名单表

如果上一步核心表基本 OK，继续在 SQL Editor 新建 Query，运行：

supabase/sql/02-install-command-center.sql

它会创建：

public.app_users register_app_owner(email) app_search_characters(jsonb) app_save_character(jsonb) app_search_parties(jsonb) app_search_versions(jsonb) app_search_resources(jsonb)

如果这里报 unique index 失败，通常是因为数据库里有重复角色名/重复 URL。先截图给我，或者临时注释对应 create unique index 那一行。

2.3 SQL 级别快速测试

select public.app_search_characters('{"keyword":"千","limit":3}'::jsonb);

能返回 JSON 数组，就说明数据库后端函数通了。

3. Supabase Auth：创建登录用户并登记 owner

3.1 创建 Auth 用户

Supabase Dashboard
→ Authentication
→ Users
→ Add user / Invite user

用你之后登录前端的邮箱创建用户。密码要记住。

3.2 登记到 app_users

打开：

supabase/sql/03-register-owner.sql

把：

select public.register_app_owner('YOUR_EMAIL@example.com');

改成你的邮箱，例如：

select public.register_app_owner('miketsusama@gmail.com');

然后在 Supabase SQL Editor 运行。运行后检查：

select * from public.app_users;

必须能看到你的邮箱，并且 is_active = true。

4. Supabase Edge Function：部署真正后端 API

后端代码位置：

supabase/functions/gameup-api/index.ts

函数名固定叫：

gameup-api

4.1 推荐方式：Supabase CLI 部署

在项目根目录打开 PowerShell / Terminal，也就是能看到 index.html 的目录。

supabase login
supabase link --project-ref YOUR_PROJECT_REF
supabase secrets set SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=YOUR_SERVICE_ROLE_KEY
supabase secrets set ALLOWED_ORIGINS=http://localhost:8000,https://YOUR_GITHUB_USERNAME.github.io
supabase functions deploy gameup-api --no-verify-jwt

YOUR_PROJECT_REF 是 Supabase URL 里这一段：

https://rubjeqnuxuvupjwyksmo.supabase.co
        ↑ 这段 rubjeqnuxuvupjwyksmo

不要把 SUPABASE_SERVICE_ROLE_KEY 写进 GitHub，也不要写进 src/config.js。它只应该存在 Supabase Edge Function secrets 里。

4.2 不想用 CLI：Dashboard 方式

Supabase Dashboard 也可以创建 Edge Function。路径通常是：

Supabase Dashboard
→ Edge Functions
→ Create a new function
→ 名字填 gameup-api
→ 把 supabase/functions/gameup-api/index.ts 的内容粘进去
→ Deploy

然后设置 Secrets：

Edge Functions
→ Secrets
→ Add secret
SUPABASE_URL = https://YOUR_PROJECT_REF.supabase.co
SUPABASE_SERVICE_ROLE_KEY = 你的 service_role key
ALLOWED_ORIGINS = http://localhost:8000,https://YOUR_GITHUB_USERNAME.github.io

Dashboard 的 UI 可能会改，但关键词就是 Edge Functions、Secrets、Deploy。

5. 前端配置：只填公开 key

打开：

apps/command-center/src/config.js

改成：

export const CONFIG = {
  SUPABASE_URL: "https://YOUR_PROJECT_REF.supabase.co",
  SUPABASE_ANON_KEY: "你的 anon / publishable key",
  EDGE_FUNCTION_NAME: "gameup-api"
};

这些值在哪里找：

Supabase Dashboard
→ Project Settings
→ API
→ Project URL
→ anon / public / publishable key

这里绝对不要填 service_role key、database password、JWT secret。

6. 本地测试

Windows：双击

scripts/serve-windows.bat

macOS：双击或终端运行

scripts/serve-macos.command

或者手动：

python -m http.server 8000

浏览器打开：

http://localhost:8000/apps/command-center/

测试顺序：

  1. 登录。
  2. 总览 → 测试后端连接。
  3. 角色 → 搜索 千 或你库里确定存在的角色。
  4. 新增一个测试角色，例如 测试角色A。
  5. 回 Supabase Table Editor 确认数据写入。
  6. 删除测试角色。

9. 常见报错怎么定位

现象原因处理
Missing SUPABASE_URL or SUPABASE_SERVICE_ROLE_KEYEdge Function secrets 没设去 Edge Functions → Secrets 设置两个值后重新部署/重试
Invalid or expired user token没登录或 token 过期前端退出再登录
Forbidden ... app_usersAuth 用户没登记到 app_users运行 supabase/sql/03-register-owner.sql
Game not found新增时游戏 code 不匹配查 select short_code, code, title from games;
column ... does not exist你微调了列名跑 supabase/sql/01-check-schema.sql 看缺哪个字段
CORS 错误ALLOWED_ORIGINS 没包含当前地址加上 http://localhost:8000 和 GitHub Pages URL
GitHub Pages 404index.html 没在根目录或 Pages 设置错确认 branch main / folder root

最后一句

这版的原则是：GitHub 只放前端，Supabase Edge Function 才是后端，Postgres 负责数据归位。这样不再让浏览器直接操作十几张表，后面扩字段、改表、加逻辑都不会把前端炸穿。数据库嘛，祖宗成法还是要讲究的：户籍归户部，兵册归兵部，别都塞进一个 Excel 格子里。🏯
